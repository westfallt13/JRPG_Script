# JRPG_Script ‚Äî Data Folder Structure

> **Note on `.uid` files:** Every `.gd` file has a corresponding `.gd.uid` sidecar generated by Godot to track the resource across the project. These are not listed individually below. Do not edit them manually.

## `data/` ‚Äî Root data directory. All static game data lives here, split into items and stats.

---

### `data/item_database/` ‚Äî Item and equipment definitions for the game's inventory system.

---

#### `data/item_database/consumables/` ‚Äî Consumable item registry and effect structure.
| File / Folder | Purpose |
|------|---------|
| [consumables.gd](data/item_database/consumables/consumables.gd) | Defines `dict_consumables` dictionary ‚Äî the top-level registry for consumable items (potions, elixirs, food, etc.). Currently empty. **Note:** has a syntax error (`inv` instead of `var`) that needs fixing before use. |
| `consumable_effect/health/` | *(Empty)* Reserved for health-restoration effect definitions. |
| `consumable_effect/mana_restoration/` | *(Empty)* Reserved for mana-restoration effect definitions. |
| `consumable_effect/stat_buffs/` | *(Empty)* Reserved for stat-buff effect definitions. |
| `consumable_type/` | *(Empty)* Reserved for consumable type classifications. |

---

#### `data/item_database/quest_items/` ‚Äî Non-equippable story and quest items.
| File | Purpose |
|------|---------|
| [guide_to_quest_items.md](data/item_database/quest_items/guide_to_quest_items.md) | Guide for creating and using quest items. |
| [quest_items.gd](data/item_database/quest_items/quest_items.gd) | Defines `dict_quest_items` dictionary with 3 placeholder entries (`quest_item_1‚Äì3`): Ancient Amulet, Enchanted Sword, Mystic Scroll ‚Äî each with a `name`, `description`, and `quest_id`. Marked as placeholder data pending replacement with real game content. |

---

#### `data/item_database/usable_items/` ‚Äî Non-consumable usable items (thrown objects, tools, etc.)
All subfolders are empty scaffolding, ready to be populated.

| Folder / File | Purpose |
|------|---------|
| [Guide_To_Usable_Items.md](data/item_database/usable_items/Guide_To_Usable_Items.md) | Guide for overall usable items. |
| `defensive_items/` | *(Empty)* Reserved for defensively-used items (shields, barriers, etc.). Includes [Guide_To_Defensive_Items.md](data/item_database/usable_items/defensive_items/Guide_To_Defensive_Items.md). |
| `offensive_items/` | *(Empty)* Reserved for offensively-used items (bombs, throwables, etc.). Includes [Guide_To_Offensive_Items.md](data/item_database/usable_items/offensive_items/Guide_To_Offensive_Items.md). |
| `support_items/` | *(Empty)* Reserved for support/utility items (keys, tools, etc.). |

---

#### `data/item_database/equipment/` ‚Äî All equippable gear the player can obtain and use.

---

#### `data/item_database/equipment/accessories/` ‚Äî Equippable accessory slots.
Each subfolder holds one `.gd` file defining a dictionary for that slot.

| Folder / File | Purpose |
|------|---------|
| [guide_to_accessories.md](data/item_database/equipment/accessories/guide_to_accessories.md) | Guide for accessory structures. |
| `accessory_effects/` | *(Empty)* Reserved for effect definitions that accessories can proc or apply. |
| `l_ring/dict.l_ring.gd` | Left ring slot ‚Äî empty scaffold. Also contains [guide_to_left_rings.md](data/item_database/equipment/accessories/l_ring/guide_to_left_rings.md). |
| `necklace/dict.necklaces.gd` | Necklace slot ‚Äî empty scaffold. Also contains [guide_to_necklaces.md](data/item_database/equipment/accessories/necklace/guide_to_necklaces.md). |
| `r_ring/dict.r_ring.gd` | Right ring slot ‚Äî empty scaffold. Also contains [guide_to_right_rings.md](data/item_database/equipment/accessories/r_ring/guide_to_right_rings.md). |

---

#### `data/item_database/equipment/armor/` ‚Äî Armor slot definitions and class system.

##### `data/item_database/equipment/armor/armor_types/` ‚Äî Individual armor slot dictionaries.
Each subfolder holds one `.gd` file defining a dictionary for that armor slot. All use the same schema: `{slot}_name`, `description`, and an `armor_class` block with `light`, `medium`, and `heavy` boolean flags (all currently `false`).

| Folder / File | Purpose |
|------|---------|
| `boots/dict.boots.gd` | Boot slot ‚Äî test item scaffold with armor class flags. |
| `bottoms/dict.bottoms.gd` | Pants/bottoms slot ‚Äî test item scaffold with armor class flags. |
| `chests/dict.chests.gd` | Chest armor slot ‚Äî test item scaffold with armor class flags. |
| `gloves/dict.gloves.gd` | Glove slot ‚Äî test item scaffold with armor class flags. |
| `helmet/dict.helmets.gd` | Helmet slot ‚Äî test item scaffold with armor class flags. |

##### `data/item_database/equipment/armor/armor_effects/`
*(Empty)* Reserved for effect definitions that armor can proc or apply (e.g., damage reflect, passive resistances). Contains [guide_to_armor_effects.md](data/item_database/equipment/armor/armor_effects/guide_to_armor_effects.md).

---

#### `data/item_database/equipment/weapons/` ‚Äî All weapon data.

##### `data/item_database/equipment/weapons/weapon_types/` ‚Äî Per-weapon-type data dictionaries.
Each subfolder holds one `.gd` file defining a dictionary for that weapon category. All use the same schema: `{type}_name`, `description`, a `weapon_class` block (`one_handed`/`two_handed` booleans), and a `weapon_effect` block (stub pointing to the `weapon_effects` dictionary).

| Folder / File | Purpose |
|------|---------|
| `axe/dict.axes.gd` | Axe weapon definitions ‚Äî test item scaffold. |
| `bow/dict.bows.gd` | Bow weapon definitions ‚Äî test item scaffold. |
| `dagger/dict.dagger.gd` | Dagger weapon definitions ‚Äî test item scaffold. |
| `greatswords/dict.greatswords.gd` | Greatsword weapon definitions ‚Äî test item scaffold. |
| `knife/dict.knives.gd` | Knife weapon definitions ‚Äî test item scaffold. |
| `staff/dict.staves.gd` | Staff weapon definitions ‚Äî test item scaffold. |
| `sword/dict.swords.gd` | Sword weapon definitions ‚Äî test item scaffold. |

##### `data/item_database/equipment/weapons/weapon_effects/` ‚Äî Per-weapon-type special effect dictionaries.
Mirrors `weapon_types/` but stores the *effects* each weapon category can proc or apply (e.g., bleed on axes, spell amplification on staves). All currently empty.

| Folder / File | Purpose |
|------|---------|
| `axe/dictionary_axe_effects.gd` | Special effects for axes (e.g., cleave, bleed). |
| `bow/dictionary_bow_effects.gd` | Special effects for bows (e.g., pierce, slow). |
| `dagger/dictionary_dagger_effects.gd` | Special effects for daggers (e.g., poison, backstab). |
| `greatswords/dictionary_greatswords_effects.gd` | Special effects for greatswords (e.g., knockback, stun). |
| `knife/dictionary_knives_effects.gd` | Special effects for knives. |
| `staff/dictionary_staves_effects.gd` | Special effects for staves (e.g., spell amplification). |
| `sword/dictionary_swords_effects.gd` | Special effects for swords. |

---

### `data/stats/` ‚Äî All character stat systems: attributes, HP/MP, mental state, and status conditions.

---

#### `data/stats/stat_types/` ‚Äî Core numerical stats that define a character's capabilities.

##### `data/stats/stat_types/attributes/`
| File | Purpose |
|------|---------|
| [attribute_stats.gd](data/stats/stat_types/attributes/attribute_stats.gd) | Defines the 6 core character attributes: `strength`, `intelligence`, `vitality`, `willpower`, `agility`, `luck` ‚Äî all defaulting to 10. Intended to be merged into party member dictionaries with a `name_` prefix for identification. |

##### `data/stats/stat_types/health_and_mana/`
| File | Purpose |
|------|---------|
| [health_and_mana.gd](data/stats/stat_types/health_and_mana/health_and_mana.gd) | Defines `current_health`, `max_health`, `current_mana`, `max_mana` (all 100). **Note:** The health/mana values are currently defined outside the dictionary brackets ‚Äî syntax error needs fixing before use. |
| `health_and_mana_helper_functions/health_and_mana_helper_functions.gd` | Placeholder for HP/MP utility functions (healing, damage application, mana consumption, etc.). Empty ‚Äî logic pending. |

---

#### `data/stats/ability_and_stability/` ‚Äî Ability definitions.

##### Reserved empty subfolders:
| Folder | Purpose |
|------|---------|
| `enemy_abilities/` | *(Empty)* Reserved for unique enemy ability definitions. |
| `magic_abilities/` | *(Empty)* Reserved for spell and magic ability data. |

---

#### `data/stats/status_conditions/` ‚Äî Debuffs and conditions that can be applied to characters.
| File | Purpose |
|------|---------|
| [status_conditions.gd](data/stats/status_conditions/status_conditions.gd) | Defines 10 status effects: **Poison, Silenced, Sleep, Burn, Cursed, Frozen, Paralyzed, Confused, Bleeding, Stunned** ‚Äî each classified as either `damage_over_time` or `status_effect`. Keyed by name for direct lookup (e.g., `status_conditions["Poison"]`). Marked as placeholder ‚Äî detailed effect values pending. |
| `status_condition_helper_functions/status_condition_helper_functions.gd` | Contains a stub `apply_status_condition(target, condition)` function. Logic not yet implemented. |

---

#### `data/stats/enemy_stats/` ‚Äî *(Empty)* Reserved for enemy-specific stat blocks and definitions.
