# JRPG_Script ‚Äî Data Folder Structure

> **Note on `.uid` files:** Every `.gd` file has a corresponding `.gd.uid` sidecar generated by Godot to track the resource across the project. These are not listed individually below. Do not edit them manually.

## `data/db_loader/` ‚Äî Central data-loading system.

All game data is authored as JSON and loaded through one autoloaded singleton. Gameplay code never reads files directly ‚Äî it asks `DbLoader`.

| File | Purpose |
|------|---------|
| [json_db.gd](data/db_loader/json_db.gd) | `class_name JsonDB` ‚Äî static JSON helpers used by every loader: `read_json`, `index_by`, `load_dir` (one item per file in a `JSON/` folder), `load_collection` (one file holding an array of effects). |
| [db_loader.gd](data/db_loader/db_loader.gd) | Autoloaded as **`DbLoader`**. Aggregates every per-folder loader, loads each category lazily and caches it. API: `get_category(name)`, `get_item(category, key)`, `get_categories([...])`, `category_names()`, `load_all()`, `reload(name)`, `clear_cache()`. |
| [equipment_effects.gd](data/db_loader/equipment_effects.gd) | `class_name EquipmentEffects` ‚Äî resolves an equipment item's named effects into full effect data. API: `for_item`, `for_item_key`, `with_effects`, `get_effect`, `effect_field`, `effects_category`, `is_linked`. |
| [guide.to.db.loader.md](data/db_loader/guide.to.db.loader.md) | Design notes for the loader. |

**How each data folder works:** every leaf folder holds a small loader `.gd` (declares a path + key field, exposes `static func get_all()`) plus a `JSON/` subfolder containing the actual data. To add an item you drop a `.json` file in `JSON/` ‚Äî no code changes. Item-type folders use one JSON file per item, keyed by a name field; `*_effects` folders use a single JSON file holding an array of effects keyed by `effect_name`.

**Effect linking:** equipment items name their effects in a per-type field ‚Äî weapons `weapon_effect`, armor `armor_effects`, accessories `accessory_effects` ‚Äî as an effect name or list of names. `EquipmentEffects` looks those names up in the matching effects collection (each weapon type resolves against its own `*_effects`; all armor shares `armor_effects`; all accessories share `accessory_effects`).

Every category below follows the same pattern: a loader `.gd` (exposes `static func get_all()`) reads the sibling `JSON/` folder. `DbLoader` category names are shown in **bold**.

##### `data/item_database/equipment/armor/armor_effects/`
